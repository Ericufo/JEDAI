# 2025-2026学年第一学期/420407-软件设计模式/期末项目要求
## 一、项目要求
1. 各候选团队需通过运用设计模式，对现有软件项目的源代码进行重构。强烈建议团队重构此前由团队成员自主开发的项目；在特殊情况下，也可考虑选择开源项目。
2. 团队需应用课堂上讲解过的**五种设计模式**（包括一种创建型模式、两种结构型模式和两种行为型模式），以及一种**课堂未覆盖的额外设计模式**。可参考维基百科上的设计模式列表（https://en.wikipedia.org/wiki/Design_Patterns），以探索更多额外的设计模式。
3. 在重构过程中，团队需探索并实践使用AI工具，用于多种用途，包括但不限于识别重构机会、提出设计方案、应用设计模式转换源代码，以及评估重构质量。
4. 团队需提交重构后的源代码和项目报告，并最终就其工作成果进行展示。


## 二、提交要求
各团队需提交一个包含以下材料的压缩包：
1. **重构后的完整源代码**：若源代码总大小超过20MB，仅需包含重构部分的代码。此外，重构后的源代码片段需通过注释清晰标记，例如“使用XXX模式重构”。
2. **一份项目报告**：需按以下顺序涵盖以下内容：
    - 软件项目名称、团队编号、所有成员的全名及学号、联系电话、电子邮箱，以及其他相关详细信息（如有）；
    - 软件项目简介，如项目背景和主要功能；
    - 重构工作的详细说明：重点标注已修改的源代码文件、重构前后的主要差异，并配合UML类图进行说明，同时解释修改原因（如解决的问题、实现的收益等）；
    - 重构过程中AI工具使用的详细说明，以及关于最佳实践、局限性、挑战与解决方案，还有其他经验教训的反思性讨论；
    - 其他可能相关或有讨论价值的补充信息。
    - 注意：报告需用英文撰写，提交文件格式为PDF。
3. **最终展示幻灯片**：第一页需呈现软件项目名称、团队编号，以及所有成员的全名及学号。
    - 注意：幻灯片需用英文制作，提交文件格式为PDF。

请将源代码（第1点）与文档（第2、3点）分别放入不同文件夹，压缩为一个ZIP文件后，通过Canvas提交。


## 三、展示要求
各团队需在10分钟内完成期末项目展示，展示结束后进行问答环节。展示需用英文进行，且团队需完成以下内容：
1. 简要介绍项目及重构的动机；
2. 明确展示并解释六种设计模式的应用：特别需通过带有清晰标题的UML类图，详细对比重构前后的源代码，说明修改原因（为何必要）及带来的收益；
3. 说明重构过程中AI工具的使用，以及关于最佳实践、局限性、挑战与解决方案，还有其他经验教训的反思性讨论；
4. 讨论重构工作中涉及的其他重要问题及相应解决方案。
